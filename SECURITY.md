# Security

Responsible disclosure: If you find a vulnerability, please contact the repository owner privately.
Replace the contact below with a real address before publishing.

Contact: security@example.com

Basic checklist for contributors:
- Do not commit secrets or credentials. Use environment variables or secret stores.
- Keep dependencies up to date; run a dependency scanner periodically.
- Validate inputs and avoid executing untrusted data.
- Limit logging of sensitive information.

Security resources and next steps are in `docs/`.
