# Contributing

Thanks for contributing! A quick guide to make clean PRs and learn Git.

Branch naming:
- `feature/<short-description>` — new feature or lesson exercise
- `fix/<short-description>` — bug fix
- `chore/<short-description>` — repo maintenance

Workflow:
1. Fork (or clone) this repo and create a branch from `main`.
2. Make focused commits with clear messages (use Conventional Commits if helpful).
3. Push your branch and open a Pull Request targeting `main`.
4. Add a clear PR description and link to the lesson or issue it solves.

PR review checklist:
- Does the change have a clear description and tests (if applicable)?
- Are secrets or credentials accidentally included? (check `.gitignore`)
- Does it follow the coding style in the repo?

If you're new to Git or PRs, follow `docs/lessons/lesson-1-git-basics.md`.
